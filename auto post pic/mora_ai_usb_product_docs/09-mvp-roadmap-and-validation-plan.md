# 09. MVP 路线图与验证计划

## 1. MVP 定义

MVP 不等于完整产品。MVP 只验证一个核心命题：

> **用户是否愿意为“插上即用的随身私人 AI 工作区”付费，并在真实跨设备/跨项目场景中持续使用。**

## 2. MVP 必须交付的体验

1. 用户拿到 U 盘。
2. 打开启动页或便携 App。
3. 激活账号和工作区。
4. 创建/导入一个项目文件夹。
5. MORA 建立索引并生成项目摘要。
6. 用户能问项目资料。
7. 用户能调用模型生成输出。
8. 用户能看到 credits 消耗。
9. 用户能安全弹出 U 盘。
10. 换一台可信电脑后可以继续使用。

## 3. 24 周 MVP 路线

### Phase 0：产品验证与原型（第 1–2 周）

目标：验证概念而不是写大量代码。

交付：

- Landing page；
- 3 分钟 demo 视频；
- Figma 原型；
- 20 个目标用户访谈；
- Founder Edition waitlist。

关键指标：

- Landing page 转化率；
- Waitlist 注册；
- 用户愿付价格；
- “U 盘形态”接受度；
- 对安全/隐私的疑问类型。

### Phase 1：核心骨架（第 3–6 周）

交付：

- U 盘目录结构；
- Start.html；
- Tauri App 骨架；
- 本地加密工作区；
- 账号激活；
- MORA Gateway mock；
- credits mock。

验收：

- Mac/Windows 可打开；
- 不需要安装复杂依赖；
- 账号可绑定设备；
- 工作区可加密保存。

### Phase 2：文件索引与项目模式（第 7–12 周）

交付：

- 项目文件夹导入；
- PDF/docx/txt/md/html 基础解析；
- SQLite metadata；
- Tantivy 全文索引；
- 简单向量索引；
- 项目摘要；
- 文件问答。

验收：

- 100 个文件内索引稳定；
- 断电/拔出恢复测试通过；
- 关键词搜索准确；
- AI 能基于项目上下文回答。

### Phase 3：模型网关与 credits（第 13–16 周）

交付：

- MORA Gateway 接入；
- 默认模型路由；
- 多模型聊天；
- 模型对比；
- credits 余额与扣费；
- API key 基础功能。

验收：

- 用户不需要配置 provider；
- 模型失败有 fallback；
- 每次调用有用量记录；
- 不出现不可解释扣费。

### Phase 4：任务包与 UX 打磨（第 17–20 周）

交付：

- Consultant Pack；
- Creator Pack；
- Recent Context 首页；
- Safe Eject；
- 诊断日志；
- 错误恢复 UX。

验收：

- 用户打开后知道下一步做什么；
- 任务包能完成 3 个真实案例；
- Safe Eject 可用；
- 用户无需理解技术术语。

### Phase 5：私测与预售（第 21–24 周）

交付：

- 20–50 个真实 U 盘；
- Founder Edition；
- 私测社区；
- 反馈系统；
- 退款/退货流程；
- 安全文档初版。

验收：

- 激活成功率；
- 第 2 次使用率；
- 跨电脑继续使用率；
- 7 日留存；
- NPS；
- 付费转化。

## 4. 核心验证指标

### 概念层指标

- Waitlist 转化率 > 5%。
- 访谈中愿意为硬件付 $49+ 的用户 > 30%。
- 用户能用自己的话复述价值主张。

### 激活层指标

- 收到 U 盘后 72 小时内激活率 > 60%。
- 首次激活成功率 > 85%。
- 第一次创建项目成功率 > 70%。

### 使用层指标

- D7 留存 > 30%。
- 用户至少创建 2 个项目空间。
- 用户至少进行 5 次项目内 AI 问答。
- 用户第二次在不同电脑打开的比例 > 20%。

### 商业层指标

- 预售退款率 < 15%。
- Pro 订阅转化 > 10%。
- credits 充值率 > 5%。
- 用户推荐率 > 10%。

## 5. 用户访谈问题

### 背景

1. 你现在一周会用几台电脑？
2. 你同时有几个客户/项目？
3. 你的资料主要散落在哪里？
4. 你现在怎么管理 prompt 和 AI 输出？
5. 你是否遇到过“换电脑找不到上下文”的情况？

### 概念测试

1. “一个装在 U 盘里的私人 AI 工作区”你第一反应是什么？
2. 你会担心什么？
3. 你愿意插到自己的电脑吗？客户电脑呢？
4. 你更想要 Web Mode 还是 Workspace Mode？
5. 如果它能自动理解项目资料，你最想让它做什么？

### 价格测试

1. $49 硬件 + $19/月是否合理？
2. 你更喜欢一次性买断、月订阅、还是 credits？
3. 哪个功能最能让你付费？
4. 你会把它送给朋友/同事吗？

## 6. MVP 不该验证什么

第一版不要试图验证：

- 企业 IT 是否允许 USB；
- 完全离线大模型是否可行；
- 所有职业都适用；
- 所有云盘都能集成；
- 团队协作是否成立；
- 高度合规行业是否可卖。

这些都是后续阶段问题。

## 7. 成功标准

MVP 成功不是“功能很多”，而是用户出现以下反应：

- “我想把我的项目资料放进去试试。”
- “这个比 ChatGPT 空白框更像我的工作区。”
- “我下次去客户那里会带着它。”
- “我愿意为它每月付费。”
- “我想给朋友看这个东西。”

如果这五个反应出现，说明方向成立。
