# 03. 目标用户与核心场景

## 1. 用户选择原则

首发用户必须同时满足四个条件：

1. **跨设备工作**：经常在不同电脑之间切换。
2. **项目上下文复杂**：不是单次聊天，而是有持续项目、客户、资料和历史。
3. **愿意为效率付费**：能理解每月几十美元换取时间节省。
4. **能接受 U 盘形态**：对实体工作区、数据随身、隐私边界有正向感知。

不建议首发纯小白、泛学生、企业大客户或高度合规行业。

## 2. 首发 Persona A：独立顾问

### 用户画像

独立顾问、自由顾问、咨询型服务提供者。他们可能做战略咨询、品牌咨询、运营咨询、财务顾问、跨境顾问、AI 顾问等。

### 痛点

- 多个客户资料混杂。
- 客户会议前需要快速回忆背景。
- 客户资料散落在邮件、云盘、Notion、PPT、本地文件夹。
- 经常使用客户会议室电脑、临时电脑、远程办公设备。
- 不希望客户资料残留在别人的电脑上。
- ChatGPT 每次都是空白对话框，不能天然理解某个客户项目。

### MORA Drive 的价值

- 每个客户一个 Project Space。
- 插上 U 盘即可打开对应客户上下文。
- AI 能总结最近变化、关键资料、待办、会议准备。
- 拔下后工作区带走，不把资料留在本机。
- 可以快速生成会议提纲、proposal、follow-up、调研摘要。

### 核心卖点

**Client work stays in your pocket.**

### 典型使用流程

1. 顾问准备 Client A 的会议。
2. 插入 MORA Drive。
3. 打开 Client A Project。
4. 首页显示“上次会议纪要、最近邮件、待确认问题、相关资料”。
5. 用户点击“Prepare meeting brief”。
6. AI 生成会议提纲和风险点。
7. 会议结束后，用户粘贴/上传纪要。
8. MORA 更新项目上下文和 follow-up。
9. 拔下 U 盘，资料随身带走。

## 3. 首发 Persona B：创意自由职业者

### 用户画像

设计师、视频剪辑师、品牌策划、文案、独立创作者、内容工作室主理人。

### 痛点

- 每个客户项目都有大量素材、参考图、脚本、客户反馈、版本记录。
- 灵感和资料分散在浏览器、Figma、云盘、聊天记录、截图和本地文件夹。
- 经常忘记之前保存过什么参考。
- AI 工具生成内容可以，但不知道项目上下文。
- 切换项目时需要重新找资料、重新解释背景。

### MORA Drive 的价值

- 打开项目文件夹时，AI 自动呈现相关素材、历史反馈、参考资料。
- Prompt、风格偏好、客户偏好随项目保存。
- 可用 AI 生成脚本、标题、广告文案、视觉参考摘要。
- 跨电脑继续工作，不重新配置。

### 核心卖点

**Your ideas, drafts, references and AI prompts travel with you.**

### 典型使用流程

1. 创作者打开“Client X Rebrand”项目。
2. MORA 自动展示品牌指南、历史反馈、参考链接、相关素材。
3. 用户问：“根据客户上轮反馈，帮我整理这次提案的方向。”
4. AI 结合项目上下文生成提案结构。
5. 用户继续让 AI 生成视觉 moodboard 文案、邮件、社媒 teaser。
6. 所有内容都沉淀在项目空间。

## 4. 第二阶段用户

### Indie Hacker / Builder

适合 Builder Kit：提供 API key、OpenAI-compatible endpoint、代码示例、模型对比和成本控制。

### 研究者 / 学生

适合 Research Kit：PDF 总结、文献问答、笔记整理。但价格敏感，首发不宜作为核心商业用户。

### 跨境电商小卖家

适合 Ecommerce Kit：商品描述、listing、客服回复、翻译、本地化文案。这个方向更偏应用包，可以在验证后单独做 SKU。

## 5. 暂不建议首发的用户

| 用户 | 暂不首发原因 |
|---|---|
| 企业 IT/政府/医院 | USB 安全审批复杂，销售周期长 |
| 律师/医疗等高度合规群体 | 需求强，但合规承诺和审计成本高 |
| 纯小白用户 | 售后成本高，难理解项目工作区价值 |
| 泛学生 | 价格敏感，容易只薅免费额度 |
| AI heavy users | 他们会拿你和 OpenRouter/TypingMind 逐项比较，品牌溢价不明显 |

## 6. 核心场景矩阵

| 场景 | 价值主张 | 对应功能 |
|---|---|---|
| 换电脑继续工作 | 我的 AI 环境跟着我 | U 盘启动、账号绑定、本地工作区 |
| 客户项目隔离 | Client A 和 Client B 不混 | Project Mode、加密空间 |
| 快速恢复上下文 | AI 知道我上次做到哪 | Recent Context、项目摘要 |
| 问我的资料 | 不用手动找文件 | 本地索引、RAG、文件问答 |
| 不想配置 AI | 打开即用 | 模型网关、默认模型、任务入口 |
| 重度模型消耗 | 额度透明且可控 | Credits 钱包、模型档位 |
| 轻量开发接入 | 直接拿 API 用 | OpenAI-compatible API |
